## 1. 参数流总览

1. ego_planner_mid360.launch 通过 include 把参数传给 advanced_param.xml。
2. advanced_param.xml 把参数注入 ego_planner_node 的私有参数空间，并做话题 remap。
3. ego_planner_node 内部模块读取参数：
   - fsm/* 由 EGOReplanFSM 读取。
   - manager/* 由 EGOPlannerManager 读取。
   - grid_map/* 由 GridMap 读取。
   - optimization/* 由 BsplineOptimizer 读取。
4. traj_server 节点单独读取 traj_server/* 参数。

## 2. ego_planner_mid360.launch 参数说明

参数定义见 [src/mission/launch/ego_planner_mid360.launch](src/mission/launch/ego_planner_mid360.launch)。

### 2.1 地图与高度相关

- map_size_x_ = 50.0
- map_size_y_ = 50.0
- map_origin_x_ = -25.0
- map_origin_y_ = -25.0
- map_size_z_ = 1.20
- ground_height = 0.3

实际生效路径：

- map_size_x_/map_size_y_/map_size_z_ 会传为 grid_map/map_size_x,y,z 并被 GridMap 读取。
- ground_height 会传为 grid_map/ground_height 并被 GridMap 读取。
- map_origin_x_/map_origin_y_ 在当前源码中虽然写入参数服务器，但 GridMap 初始化原点时没有读取它们，而是直接按 map_size 计算：
  - map_origin = (-x_size/2, -y_size/2, ground_height)
  - 参考 [src/ego-planner-swarm/src/planner/plan_env/src/grid_map.cpp#L130](src/ego-planner-swarm/src/planner/plan_env/src/grid_map.cpp#L130)

结论：

- map_origin_x_ 与 map_origin_y_ 在当前代码版本属于“已配置但未实际参与地图原点计算”。

### 2.2 传感器/里程计话题相关

- odometry_topic = /mavros/local_position/odom
- cloud_topic = /cloud_registered
- scan_topic = 空字符串

实际生效路径：

- odometry_topic 会 remap 到 ego_planner_node 的 odom_world 与 grid_map/odom。
- cloud_topic 会 remap 到 grid_map/cloud。
- scan_topic 虽有 remap 项，但当前 GridMap 未订阅 grid_map/scan，因此不生效。

### 2.3 运动学与规划范围

- max_vel = 1.0
- max_acc = 1.0
- planning_horizon = 5.0

实际生效路径：

- manager/max_vel, manager/max_acc 被 EGOPlannerManager 读取，决定轨迹生成与可行性检查基础约束。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L19](src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L19)
- optimization/max_vel, optimization/max_acc 被 BsplineOptimizer 读取，参与优化约束项。
  - 参考 [src/ego-planner-swarm/src/planner/bspline_opt/src/bspline_optimizer.cpp#L17](src/ego-planner-swarm/src/planner/bspline_opt/src/bspline_optimizer.cpp#L17)
- fsm/planning_horizon 被 EGOReplanFSM 读取，用于选取局部目标点（Local Target）和过滤远处编队轨迹。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L22](src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L22)
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L915](src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L915)

### 2.4 优化权重

- lambda_smooth = 0.3
- lambda_collision = 2.0
- lambda_feasibility = 0.1
- lambda_fitness = 1.3
- swarm_clearance = 0.5
- dist = 0.20

实际生效路径：

- 由 BsplineOptimizer::setParam 读取。
  - 参考 [src/ego-planner-swarm/src/planner/bspline_opt/src/bspline_optimizer.cpp#L10](src/ego-planner-swarm/src/planner/bspline_opt/src/bspline_optimizer.cpp#L10)
- 主要作用：
  - lambda_smooth: 平滑项权重。
  - lambda_collision: 距障项与终端项核心权重。
  - lambda_feasibility: 可行性项（速度/加速度等）权重。
  - lambda_fitness: 精修阶段拟合项权重。
  - dist: 与障碍物最小安全距离门限（cps clearance）。
  - swarm_clearance: 编队间安全距离。

说明：

- swarm_clearance 不只用于优化，还用于在线碰撞检查。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L702](src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L702)
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L368](src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L368)

### 2.5 飞行模式与实验模式

- flight_type = 1
- realworld_experiment = true

实际生效路径：

- flight_type 由 FSM 读取：
  - 1 表示手动目标（订阅 /ego_planner/goal）
  - 2 表示预设路径点
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L16](src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L16)
- realworld_experiment 会影响触发逻辑：
  - true 时初始 have_trigger 为 false，需要触发机制配合。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L26](src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L26)

### 2.6 traj_server 参数

- traj_server/time_forward = 1.0
- traj_server/limit_yaw_change = 0.0

实际生效路径：

- traj_server/time_forward 与 traj_server/limit_yaw_change 在 traj_server 节点启动时读取。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/traj_server.cpp#L312](src/ego-planner-swarm/src/planner/plan_manage/src/traj_server.cpp#L312)

注意：

- traj_server 内部里程计订阅是写死的 /mavros/local_position/odom，并没有使用 advanced_param.xml 中的 odometry_topic 参数。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/traj_server.cpp#L294](src/ego-planner-swarm/src/planner/plan_manage/src/traj_server.cpp#L294)

## 3. advanced_param.xml 参数详解

参数定义见 [src/mission/launch/advanced_param.xml](src/mission/launch/advanced_param.xml)。

### 3.1 输入参数 arg 层

这些 arg 分两类：

1. 上层 launch 显式传入
2. 本文件自身 default

在 mid360 场景中，常见默认值生效的是：

- camera_pose_topic
- depth_topic
- cx/cy/fx/fy
- point_num 与 point0..3
- uav_id

### 3.2 remap 话题层

主要 remap 如下：

- odom_world -> odometry_topic
- planning/bspline -> /planning/bspline
- planning/data_display -> /planning/data_display
- planning/broadcast_bspline_from_planner <-> /broadcast_bspline
- grid_map/odom -> odometry_topic
- grid_map/cloud -> cloud_topic
- grid_map/scan -> scan_topic
- grid_map/pose -> camera_pose_topic
- grid_map/depth -> depth_topic

影响说明：

- 规划输出、可视化输出、广播轨迹都会受这些 remap 直接影响。
- 如果话题名不匹配，常见现象是：无地图更新、无局部重规划、控制指令不更新。

### 3.3 fsm/* 参数

由 EGOReplanFSM 读取：

- fsm/flight_type
- fsm/thresh_replan_time
- fsm/thresh_no_replan_meter
- fsm/planning_horizon
- fsm/emergency_time
- fsm/realworld_experiment
- fsm/fail_safe
- fsm/waypoint_num 与 fsm/waypoint{i}_{x,y,z}

对应读取代码：

- [src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L16](src/ego-planner-swarm/src/planner/plan_manage/src/ego_replan_fsm.cpp#L16)

调参直觉：

- thresh_replan_time 越小，重规划越频繁，算力压力更大。
- thresh_no_replan_meter 越小，越接近终点才停止重规划。
- emergency_time 越大，遇到临近碰撞时更偏向急停。
- fail_safe 建议保持 true，避免深度/里程计异常时继续执行风险轨迹。

### 3.4 grid_map/* 参数

由 GridMap 读取：

- 分辨率与尺寸：resolution, map_size_x,y,z
- 局部更新窗口：local_update_range_x,y,z
- 障碍膨胀与边界：obstacles_inflation, local_map_margin
- 高度相关：ground_height, virtual_ceil_height, visualization_truncate_height
- 相机内参：cx,cy,fx,fy
- 深度滤波：use_depth_filter, depth_filter_tolerance,maxdist,mindist,margin,k_depth_scaling_factor,skip_pixel
- 概率更新：p_hit,p_miss,p_min,p_max,p_occ,min_ray_length,max_ray_length
- 其他：show_occ_time, pose_type, frame_id, odom_depth_timeout(默认读取)

对应读取代码：

- [src/ego-planner-swarm/src/planner/plan_env/src/grid_map.cpp#L52](src/ego-planner-swarm/src/planner/plan_env/src/grid_map.cpp#L52)

关键机制：

- virtual_ceil_height 如果超过 ground_height + map_size_z，会被自动截断。
  - 参考 [src/ego-planner-swarm/src/planner/plan_env/src/grid_map.cpp#L124](src/ego-planner-swarm/src/planner/plan_env/src/grid_map.cpp#L124)
- map_origin 实际按 map_size 与 ground_height重算，忽略 grid_map/map_origin_x,y。

### 3.5 manager/* 参数

由 EGOPlannerManager 读取：

- manager/max_vel
- manager/max_acc
- manager/max_jerk
- manager/feasibility_tolerance
- manager/control_points_distance
- manager/planning_horizon
- manager/use_distinctive_trajs
- manager/drone_id

对应代码：

- [src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L19](src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L19)

影响说明：

- control_points_distance 决定轨迹控制点离散尺度，影响平滑与可解性。
- use_distinctive_trajs 打开后会做多候选轨迹尝试，成功率更高但耗时增加。

### 3.6 optimization/* 参数

由 BsplineOptimizer 读取：

- optimization/lambda_smooth
- optimization/lambda_collision
- optimization/lambda_feasibility
- optimization/lambda_fitness
- optimization/dist
- optimization/swarm_clearance
- optimization/max_vel
- optimization/max_acc
- optimization/order

对应代码：

- [src/ego-planner-swarm/src/planner/bspline_opt/src/bspline_optimizer.cpp#L10](src/ego-planner-swarm/src/planner/bspline_opt/src/bspline_optimizer.cpp#L10)

补充：

- optimization/order 在当前代码中确实会参与多个循环边界与采样区间计算，不是摆设。

### 3.7 bspline/* 参数

advanced_param.xml 设置了：

- bspline/limit_vel
- bspline/limit_acc
- bspline/limit_ratio

当前版本状态：

- 在当前 planner 主流程中没有看到对这三个 ROS 参数的读取。
- 实际物理约束主要由 manager/max_vel, manager/max_acc 与 feasibility_tolerance 决定。
  - 参考 [src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L299](src/ego-planner-swarm/src/planner/plan_manage/src/planner_manager.cpp#L299)

## 4. 当前文件中“常被误解/容易踩坑”的参数

1. map_origin_x_ / map_origin_y_
- 在本版本 GridMap 初始化中未用于 map_origin。

2. scan_topic
- 当前 GridMap 没有 scan 订阅，设置后通常不会产生效果。

3. bspline/limit_vel, bspline/limit_acc, bspline/limit_ratio
- 当前主流程基本未读取。

4. traj_server 与 odometry_topic
- traj_server 的里程计订阅写死，不随 advanced_param.xml 的 odometry_topic 自动变化。

5. realworld_experiment
- 会影响触发逻辑，与飞行模式共同决定是否等待触发消息。

## 5. mid360 现有参数组合的行为总结

在当前 mid360 配置下：

- 使用手动目标模式 flight_type=1。
- 地图高度层约束来自 ground_height=0.3 与 map_size_z=1.2。
- 速度加速度上限均为 1.0，整体偏保守稳定。
- 轨迹优化权重偏向避障与可行性平衡（collision 2.0, smooth 0.3, feasibility 0.1）。
- traj_server 将根据 B 样条生成位置/速度/加速度命令并发布到 /position_cmd。
