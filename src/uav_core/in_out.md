

Ego-Planner 作为一个独立节点（通常是 `ego_planner_node`），其核心交互接口如下：

### 1. 外部输入 (Inputs - 喂给 Ego 的数据)
| 话题名称 (常用名)             | 消息类型                    | 作用                                                                                                             | 必须性 |
| :---------------------------- | :-------------------------- | :--------------------------------------------------------------------------------------------------------------- | :----- |
| `/mavros/local_position/odom` | `nav_msgs/Odometry`         | **定位基础**。告诉 Ego 飞机当前的 XYZ 位置、速度和姿态。                                                         | 是     |
| `/cloud_registered`           | `sensor_msgs/PointCloud2`   | **障碍物地图**。由雷达或深度相机生成的 3D 点云。                                                                 | 是     |
| `/move_base_simple/goal`      | `geometry_msgs/PoseStamped` | **任务目标**。主控 FSM 发出的终点坐标。发送一次，触发一次规划。                                                  | 是     |
| `/traj_start_trigger`         | `geometry_msgs/PoseStamped` | **启动信号**。当 `realworld_experiment` 设为 true 时，必须给这个话题发个包（内容任意），Ego 才会由待机转为工作。 | 是     |

### 2. 内部输出 (Outputs - Ego 算出的结果)
| 话题名称 (常用名)        | 消息类型                         | 作用                                                                           | 接收者                |
| :----------------------- | :------------------------------- | :----------------------------------------------------------------------------- | :-------------------- |
| `/position_cmd`          | `quadrotor_msgs/PositionCommand` | **核心控制量**。包含：期望位置(P)、期望速度(V)、期望加速度(A)、期望偏航(Yaw)。 | 你的 `ego_controller` |
| `/planning/bspline`      | `quadrotor_msgs/Bspline`         | **样条曲线原始数据**。主要用于多机编队同步，单机忽略。                         | N/A                   |
| `/planning/data_display` | `visualization_msgs/Marker`      | **RViz 可视化**。显示规划出的绿色轨迹线、红色的预测碰撞点等。                  | RViz                  |

## 二、 关键参数详解与调试 (Parameter Tuning)

参数主要分布在 `advanced_param.xml` 中，按功能分为四组：

### 1. 运动学限制 (Dynamics Limits)
**决定飞机飞行的物理边界，乱飞时优先调这里。**

*   **`max_vel` (m/s)**: 轨迹允许的最大速度。
    *   *调试：* 比赛初期建议 `1.0`。如果飞机跟踪轨迹时出现猛烈晃动，说明速度超过了飞控处理能力，调小它。
*   **`max_acc` (m/s²)**: 允许的最大加速度。
    *   *调试：* 建议 `1.0 ~ 2.0`。设置太大会导致电机瞬间满转，引起姿态失稳；太小会导致避障不及时。
*   **`planning_horizon` (m)**: 局部规划的感知距离。
    *   *调试：* 建议设为雷达有效距离的 0.8 倍。如果设太长，飞机会因为看不到远处的点云而频繁报错。

### 2. 地图与碰撞参数 (Grid Map & Collision)
**决定飞机对障碍物的敏感度（怕不怕撞）。**

*   **`resolution` (m)**: 地图网格大小。
    *   *调试：* 建议 `0.1`。设太小（如0.05）CPU会爆；设太大（如0.3）飞机会看不见细柱子。
*   **`obstacles_inflation` (m)**: 障碍物膨胀半径。
    *   *调试：* **核心参数**。应设为 `(机体物理半径 + 留白安全余量)`。建议 `0.2 ~ 0.4`。如果飞机过不去窄门，调小它；如果总是擦边撞墙，调大它。
*   **`dist` (m)**: 轨迹优化时的避障安全门限。
    *   *调试：* 配合膨胀半径使用。通常设为 `0.2` 左右。

### 3. 轨迹优化权重 (Optimization Weights)
**决定轨迹的形状（是走直线还是走丝滑圆弧）。**

*   **`lambda_smooth`**: 平滑度权重。
    *   *调试：* 增大它会让轨迹更直、拐弯弧度更大；减小它会让轨迹更紧凑，但可能会有锯齿状抖动。
*   **`lambda_collision`**: 避障权重。
    *   *调试：* **最关键**。如果飞机明明规划出轨迹了却还是撞墙，大幅调高这个值（例如从 2.0 调到 5.0）。
*   **`lambda_feasibility`**: 可行性权重（速度加速度约束）。
    *   *调试：* 保持默认。如果飞机总是规划出“物理上飞不出来”的急转弯，调大它。

### 4. 状态机逻辑参数 (Internal FSM)
*   **`thresh_replan_time` (s)**: 强制重规划频率。
    *   *调试：* 建议 `0.1` (即 10Hz)。如果遇到动态障碍物，减小这个值增加重规划频率。
*   **`realworld_experiment` (bool)**:
    *   *调试：* 设为 `true`。它会让 Ego 变得严谨，只有在收到 `trigger` 信号且 Odom 信号稳定时才开始规划。
